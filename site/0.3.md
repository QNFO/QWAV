### MISSING 6: Math Rendering — Clarify Implementation

**Current prompt says:**
> "ALWAYS use the `.math-serif` CSS class for physical variables... Variables MUST be in italics (serif). Units (e.g., K, meV, GHz, W) MUST be upright (sans-serif) within that span."

**The problem:** This CSS-class approach conflicts with how MathJax/KaTeX handle math rendering. If the site uses MathJax (recommended for a research-heavy site), LaTeX `$...$` blocks automatically render variables in italic and `\mathrm{}` renders units upright. A custom CSS class may override MathJax styles, causing visual bugs.

**Suggested replacement:**

> **MATH RENDERING — USE MathJax OR KaTeX:**
>
> All mathematical notation must be rendered with MathJax 3.x or KaTeX (prefer MathJax for accessibility features). Use standard LaTeX delimiters: `$...$` for inline math, `$$...$$` for display math.
>
> - Variables: rendered automatically in italic serif (standard LaTeX behavior). Example: `$d$`, `$p_c$`, `$E_{\text{barrier}}$`.
> - Units: use `\mathrm{}` for upright roman per ISO 80000-2. Example: `$\mathrm{K}$`, `$\mu\mathrm{W}$`, `$\mathrm{GHz}$`.
> - Constants: use upright roman. Example: `$\mathrm{e}$` for Euler's number, `$\mathrm{i}$` for imaginary unit (unless project convention uses italic).
> - Multi-letter variables: use `\text{}` or `\mathit{}` as appropriate.
> - Numbers with thousands separators: use `{,}` thin space. Example: `$20{,}000$`.
> - Do NOT use custom CSS classes to style math characters. Let MathJax/KaTeX handle typography. The only CSS applied to math containers should control font size and color, not italic/upright styling.
>
> **Math must render correctly on mobile.** Test that inline math does not overflow containers and that display math does not require horizontal scrolling.
